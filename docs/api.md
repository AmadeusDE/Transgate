# Transgate REST API Documentation

Transgate is a powerful translation aggregator that combines results from multiple translation providers (Google, DeepL, and various LLMs) and provides a merged, high-quality translation along with linguistic context.

## Base URL

By default, the Transgate API runs on:
`http://localhost:8080`

## Endpoints

### 1. Translate Text
Returns a merged translation from multiple providers, optionally including a linguistic explanation.

- **URL**: `/translate`
- **Method**: `POST`
- **Content-Type**: `application/json`

#### Request Body
| Field | Type | Description |
| :--- | :--- | :--- |
| `text` | string | **Required**. The text to translate. |
| `source_lang` | string | *Optional*. ISO 639-1 code of the source language. If omitted, the language is automatically detected. |
| `target_lang` | string | *Optional*. ISO 639-1 code of the target language. Defaults to `en`. |
| `explain` | boolean | *Optional*. If `true`, includes a detailed linguistic explanation of the translation. |

#### Example Request
```bash
curl -X POST http://localhost:8080/translate \
  -H "Content-Type: application/json" \
  -d '{
    "text": "こんにちは世界",
    "target_lang": "en",
    "explain": true
  }'
```

#### Response Body
Returns an `AggregatedResult` object.

| Field | Type | Description |
| :--- | :--- | :--- |
| `original` | string | The original input text. |
| `source_lang` | string | The detected or provided source language code. |
| `target_lang` | string | The target language code. |
| `merged` | string | The final merged translation generated by the configured LLM merger. |
| `explanation` | string | A concise linguistic explanation (only if `explain: true` was requested). |
| `providers` | object | A map of individual provider results, where the key is the provider name. |

**Provider Result Object:**
| Field | Type | Description |
| :--- | :--- | :--- |
| `text` | string | The translation provided by this specific provider. |
| `alternatives` | array | List of alternative translations (if supported by the provider, e.g., DeepL). |

---

### 2. Detect Language
Detects the language of the provided text using a weighted scoring system involving Lingua-go and configured LLMs.

- **URL**: `/detect`
- **Method**: `POST`
- **Content-Type**: `application/json`

#### Request Body
| Field | Type | Description |
| :--- | :--- | :--- |
| `text` | string | **Required**. The text for which to detect the language. |

#### Example Request
```bash
curl -X POST http://localhost:8080/detect \
  -H "Content-Type: application/json" \
  -d '{
    "text": "C'est la vie"
  }'
```

#### Response Body
Returns a JSON object where keys are ISO 639-1 language codes and values are the calculated confidence scores (weights).

```json
{
  "fr": 0.95,
  "en": 0.02,
  ...
}
```

---

## Error Handling
The API returns standard HTTP status codes:
- `200 OK`: Request successful.
- `400 Bad Request`: Invalid JSON payload.
- `405 Method Not Allowed`: Incorrect HTTP method used.
- `500 Internal Server Error`: An error occurred during translation or detection.
